# Support

WeaveRx is open-source maintainer tooling. Support is **best-effort** from volunteers and maintainers.

## How to get help

| Need | Where |
| ---- | ----- |
| Bug or unexpected behavior | [Bug report](.github/ISSUE_TEMPLATE/bug_report.yml) |
| Feature idea | [Feature request](.github/ISSUE_TEMPLATE/feature_request.yml) |
| General WeaveRx feedback | [Triage feedback template](.github/ISSUE_TEMPLATE/triage_feedback.yml) |
| Security vulnerability | [SECURITY.md](SECURITY.md) — private disclosure only |
| Ethics / responsible use | [ETHICS.md](ETHICS.md) |

Before opening an issue:

1. Run the latest [release](https://github.com/FratresMedAI/WeaveRx/releases) or `master`.
2. Try `--mock` to reproduce without API keys when possible.
3. Redact tokens, patient data, and PHI from commands and logs.

## Response expectations

- Issues are triaged as time allows; there is no SLA for community support.
- Critical security reports are prioritized per [SECURITY.md](SECURITY.md).
- Pull requests with tests and clear descriptions are reviewed faster than drive-by issues.

## What we cannot help with

- Clinical decision-making or deployment in regulated care settings
- Legal or compliance sign-off for your organization's PHI policies
- Debugging your LLM provider billing, quotas, or account access
